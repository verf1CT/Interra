# App Store — чек-лист готовности к публикации

Статус: подготовка во время ожидания активации Apple Developer аккаунта.
Отметки: ✅ готово · 🟡 подготовлено, нужно подключить · ⬜ предстоит.

## Технические артефакты
- ✅ Иконка приложения (логотип Интерры, 1024×1024, без альфа-канала).
- ✅ Нативная заставка (flutter_native_splash).
- 🟡 **Privacy Manifest** — `app/ios/Runner/PrivacyInfo.xcprivacy` создан.
  Подключить: в Xcode перетащить файл в target Runner, отметить Target Membership
  (Copy Bundle Resources). Без этого App Store отклонит сборку.
- 🟡 **Push (APNs)** — см. `docs/IOS_PUSH_SETUP.md` (нужен платный аккаунт).
- ⬜ `ITSAppUsesNonExemptEncryption` в Info.plist = `false` (используется только
  стандартный HTTPS) — добавить, чтобы не заполнять экспортный опросник при каждой сборке.

## Метаданные в App Store Connect (когда аккаунт активен)
- ⬜ Название, подзаголовок, описание, ключевые слова (рус.).
- ⬜ Категория: «Утилиты» или «Бизнес».
- ⬜ Возрастной рейтинг (анкета) — ожидается 4+.
- ⬜ **Скриншоты**: 6.7″ (iPhone 15/16/17 Pro Max) и 6.5″ — минимум по 2–3 экрана
  (вход, кабинет, настройки). Можно снять с текущих телефонов.
- ⬜ **URL политики конфиденциальности** — обязателен (см. ниже).
- ⬜ Контакт поддержки (email/телефон Интерры), маркетинговый URL (сайт провайдера).

## Политика конфиденциальности
- ⬜ Опубликовать страницу (например, `https://interra.ru/app-privacy`) с описанием:
  какие данные собираются (телефон, идентификатор устройства для push), зачем
  (доступ к ЛК, уведомления о балансе/тарифе), что не передаётся третьим лицам и
  не используется для рекламы. Совпадает с `PrivacyInfo.xcprivacy`.

## Соответствие правилам App Review (риски для WebView-приложения)
- Приложение — обёртка над веб-кабинетом UTM5. Чтобы пройти ревью как «не просто
  сайт», у нас уже есть нативная ценность: биометрический вход, push-уведомления,
  нативный экран входа/регистрации, вынос tel/mailto в системные приложения.
  При подаче — подчеркнуть это в заметках для ревьюера.
- ⬜ Тестовый аккаунт для ревьюера — подготовить рабочий демо-логин в ЛК и указать
  его в App Review Information (без рабочего входа ревью отклонят).
