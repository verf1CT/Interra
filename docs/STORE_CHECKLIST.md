#### Демо-аккаунт: как именно (вход по SMS-коду)

Подвох: вход — телефон + одноразовый SMS-код через биллинг UTM5 (`bbb`).
Модератор Apple/Google не может получить SMS на чужой номер, поэтому обычный
«логин/пароль» не подойдёт, а без рабочего входа заявку отклонят.

- [ ] **Тестовый абонент с фиксированным кодом** (правильный путь). Попросить
      админов биллинга завести служебного абонента, у которого:
  - есть выделенный номер телефона;
  - на запрос кода биллинг принимает **фиксированный OTP** (например, всегда
    `0000`) вместо реальной SMS;
  - на счёте есть баланс/данные, чтобы кабинет отрисовался (пустой ЛК ревьюер
    примет за «не работает»).
- [ ] Проверить, что этот номер реально логинится в приложении **до** подачи.
- [ ] Если бэкдор в биллинге невозможен — запасной вариант: видео прохода по
      приложению в заметках ревьюеру (работает хуже, Apple часто всё равно
      просит доступ).

Куда вписать при подаче:
- **App Store Connect** → *App Review Information* → **Sign-In required** →
  `User Name` = номер, `Password` = фиксированный код (или в *Notes*). В *Notes*
  описать флоу: «ввести номер X → "Получить код" → ввести код Y».
- **Google Play Console** → *App content* → **App access** → *restricted* →
  те же учётные данные и инструкция.

В заметках ревьюеру подчеркнуть нативную ценность (биометрия, push, диагностика
сети, сканер Wi-Fi, нативный баланс/виджеты) — чтобы пройти правило Apple 4.2
«не просто сайт». Подробнее — в `docs/APP_STORE_READINESS.md`.
