# ЛК Интерра

Мобильное приложение «Личный кабинет» для абонентов интернет-провайдера
**Интерра** (`interra.ru`, г. Первоуральск). Один код на iOS и Android (Flutter).
Показывает веб-кабинет, нативно подтягивает баланс, шлёт push-уведомления и даёт
полезные инструменты (диагностика сети, «кто в моём Wi-Fi», виджеты баланса).

## Что это и как устроено (простыми словами)

Приложение — это личный кабинет провайдера в телефоне. Внутри три независимых части:

1. **Приложение ↔ биллинг** — вход по номеру телефона и SMS, показ кабинета и
   баланса. Идёт напрямую к биллингу провайдера, **наш сервер тут не участвует**.
2. **Приложение → наш сервер** — приложение сообщает серверу «вот моё устройство,
   шлите мне уведомления». Только для этого сервер и нужен.
3. **Наш сервер → приложение** — оператор из веб-панели рассылает уведомления
   (баланс, тариф, сообщения), они прилетают на телефоны через Firebase.

```
┌─────────────────────────────┐   регистрация push-токена    ┌───────────────────────────┐
│  Приложение (app/)           │ ───────────────────────────▶ │  Сервер (server/)          │
│  • кабинет в WebView         │                              │  • база устройств (SQLite) │
│  • нативный баланс           │ ◀──── push (через Firebase)  │  • рассылка уведомлений    │
│  • диагностика / Wi-Fi       │                              │  • веб-панель оператора    │
└─────────────────────────────┘                              └───────────────────────────┘
        │  прямые запросы к биллингу (HTTPS)
        ▼
   биллинг провайдера (вход по SMS, страница кабинета, баланс)
```

## Как работает приложение (подробнее)

- **Вход** — по номеру телефона и SMS-коду через биллинг UTM5. Приложение хранит
  выданный токен в защищённом хранилище (Keychain / EncryptedSharedPreferences).
- **Кабинет** показывается в WebView; поверх вживляются фирменные стили и тёмная
  тема, а баланс дополнительно парсится и выводится нативно (шапка, виджеты).
- **Безопасность** — опциональный вход по Face ID / отпечатку / коду, заслонка
  приватности в переключателе задач, сертификат-пиннинг для своих хостов.
- **Инструменты** — пошаговая диагностика сети и сканер «Устройства в моём Wi-Fi».
- **Уведомления** — оператор из веб-панели сервера рассылает их всем, сегменту
  (по категориям) или конкретному абоненту. Плюс приложение подписано на общий
  FCM-топик `all` — это канал массовой рассылки «на все Android» напрямую через
  Firebase, без веб-панели и таргетинга по сегментам (см. ниже).

Подробнее — в [`app/README.md`](app/README.md) и [`server/README.md`](server/README.md).

### Как разослать уведомление на все Android

Не требует поднятого сервера — только `serviceAccountKey.json` проекта Firebase:

```bash
cd server
node scripts/send-test-push.js --topic all "Заголовок" "Текст"   # всем, кто на топике
node scripts/send-test-push.js --token <FCM_TOKEN> "Тест" "Текст" # точечно, на одно устройство
```

Долетает до устройств, где стоит сборка с подпиской на топик (см. `PushService`) и
приложение хотя бы раз открывали. Альтернатива — Firebase Console → Messaging →
**Target = Topic `all`**. Свежие APK/AAB — во вкладке **Releases** (тег `build-<N>`).

## Что нужно для запуска в магазины (кратко)

- **Сервер уведомлений** — поднять `server/` на `push.interra.ru` (на серверных
  админах). Пошаговая инструкция: [`server/ADMIN.md`](server/ADMIN.md)
  (кратко — [`docs/DEPLOY.md`](docs/DEPLOY.md)).
- **Android (Google Play):** релизный ключ уже есть (`app/android/key.properties`
  + `upload-keystore.p12`, оба вне git). **Не потерять ключ и пароль.**
- **iOS (App Store):** при активации Apple Developer включить Push и APNs-ключ.
  См. [`docs/IOS_PUSH_SETUP.md`](docs/IOS_PUSH_SETUP.md).
- **Аккаунты магазинов** — от админов. Полный список задач: [`docs/STORE_CHECKLIST.md`](docs/STORE_CHECKLIST.md).

## Быстрый старт (для разработчика)

```bash
# приложение
cd app && flutter pub get && flutter run          # запуск на устройстве/эмуляторе
flutter analyze lib test && flutter test          # проверки
flutter build apk --release                        # Android-сборка (нужен Android SDK)
flutter build ios  --release                       # iOS-сборка (нужен macOS + Xcode)

# бэкенд (нужен Node 18–20: better-sqlite3 не собирается под Node 26+)
cd server && npm install && npm start              # без ключа Firebase — режим dry-run
```

> Локальный запуск проще всего через Docker: `cd server && ./deploy.sh` (Node 20
> внутри образа). Напрямую — версией Node 18–20, иначе `better-sqlite3` падает.

Установка окружения — [`docs/INSTALL_FLUTTER.md`](docs/INSTALL_FLUTTER.md),
[`docs/INSTALL_MACOS_IOS.md`](docs/INSTALL_MACOS_IOS.md). Деплой сервера —
