# Приложение «ЛК Интерра» (Flutter)

Личный кабинет абонента: веб-кабинет UTM5 в WebView + нативные баланс,
уведомления, безопасность и платформенные фишки iOS/Android. Единый код на обе
платформы, светлая и тёмная темы в фирменных цветах Интерры (синий `#3A96D6`,
оранжевый `#F77D31`), фирменный шрифт Manrope.

## Как это работает

### 1. Вход (схема `bbb` биллинга UTM5)

Отдельного «API входа» у UTM5 нет — есть штатный CGI `stat.interra.ru/cgi-bin/utm5/bbb`.
Регистрация одноразовая, дальше живём по токену:

1. `cmd=set&app={15 случайных цифр}` → биллинг возвращает **24-значный токен приложения**.
2. `cmd=get&num={телефон}&app={токен}` → биллинг шлёт абоненту **SMS с кодом**.
3. `cmd=set&num={код}&app={токен}` → подтверждение; приложение занесено в базу.
4. `cmd=open&app={токен}` → при **каждом** открытии выдаёт свежую ссылку на кабинет
   (`?login=…`, живёт ~30 минут). Из неё собирается адрес страницы «Основная информация».

Токен и подтверждённый телефон хранятся в защищённом хранилище системы (Keychain /
EncryptedSharedPreferences). Коды ответа `0`/`1` разбираются как конкретные ошибки
(регистрация потеряна / телефон отвязан) с понятным текстом пользователю.

### 2. Кабинет и баланс

Кабинет рендерится в `WebView` (`stat.interra.ru`). Поверх страницы UTM5:

- **вживляются фирменные стили** (типографика, карточки, поля, тёмная тема) — только
  визуально, вёрстку и формы не трогаем;
- добавляется **pull-to-refresh** (нативного у UTM нет);
- **баланс и номер счёта парсятся** из текста страницы и показываются нативно —
  в чипе шапки, виджетах, на часах, через Siri;
- **снимок страницы кэшируется** — при отсутствии сети кабинет открывается офлайн
  (текст читается, баланс из кэша);
- если сессия протухла и UTM показал форму входа — ссылка **молча перезапрашивается**.

### 3. Безопасность

- **Замок**: Face ID / отпечаток и/или **код-пароль** (4 цифры, хранится как
  SHA-256 с солью; антиперебор — пауза после 5 ошибок). Настраиваемая задержка
  «запрашивать вход» (сразу / 1 мин / 30 мин / никогда).
- **Заслонка приватности** в переключателе задач — вместо баланса система снимает
  брендовую заставку.
- **Сертификат-пиннинг** (корни Let's Encrypt) для запросов к своим хостам —
  защита от MITM на чужом Wi-Fi.

### 4. Инструменты и уведомления

- **Диагностика сети** — шаги «интернет → DNS → сайт → биллинг» с вердиктом, где
  проблема и что делать.
- **Устройства в сети** — сканер локального Wi-Fi (TCP-опрос подсети + имена по
  mDNS/Bonjour): «кто подключён к моему роутеру».
- **Push** — Firebase Cloud Messaging; категории (аварии / баланс / новости)
  уходят на бэкенд как сегменты, оператор шлёт таргетированно.

## Структура `lib/`

| Путь | Назначение |
|------|------------|
| `main.dart` | Точка входа: UI показывается сразу, Firebase/телеметрия/push/версия поднимаются в фоне (чтобы не блокировать первый кадр) |
| `config.dart` | Адреса биллинга/бэкенда, контакты поддержки |
| `theme.dart` | Фирменная палитра, тема Material 3, контроллер темы (Авто/Светлая/Тёмная) |
| `utils/phone.dart` | Нормализация и форматирование телефона (покрыто тестами) |

### Экраны (`lib/screens/`)

| Файл | Экран |
|------|-------|
| `register_screen.dart` | Вход: телефон → SMS-код (антиспам-кулдаун 60 с, автоподстановка кода из SMS) |
| `webview_screen.dart` | Кабинет в WebView: авто-получение ссылки, стили/тёмная тема, офлайн-кэш, pull-to-refresh, чип баланса, восстановление сессии |
| `settings_screen.dart` | Аккаунт, тема, категории уведомлений, замок (биометрия/PIN), задержка автоблокировки, выход |
| `support_screen.dart` | Связь с провайдером (звонок, Telegram, ВКонтакте, помощь), диагностика, устройства в сети |
| `diagnostics_screen.dart` | Пошаговая диагностика сети с вердиктом |
| `lan_devices_screen.dart` | Список устройств в Wi-Fi (иконка, имя, IP) |
| `biometric_gate.dart` | Замок поверх всего приложения (Face ID/отпечаток + PIN) |
| `pin_setup_screen.dart` | Установка/смена/проверка код-пароля |

### Сервисы (`lib/services/`)

| Файл | Назначение |
|------|------------|
| `billing_api.dart` | Штатный API UTM5 `bbb` (регистрация по SMS, ссылка на кабинет) |
| `secure_http.dart` | HTTP-клиент с **пиннингом** корней Let's Encrypt для своих хостов |
| `auth_store.dart` | Токен и телефон в Keychain / EncryptedSharedPreferences |
| `balance_store.dart` | Распарсенный баланс: кэш, форматирование «1 846,03 ₽», отдача в виджеты/часы |
| `page_cache.dart` | Офлайн-снимок кабинета (HTML в файл + baseUrl), чистится при выходе |
| `push_service.dart` | FCM: разрешения, локальные уведомления на переднем плане, регистрация токена |
| `notify_prefs.dart` | Категории уведомлений (сегменты для бэкенда) |
| `api_client.dart` | Регистрация/удаление устройства на бэкенде |
| `biometric.dart` | Биометрия + льготный период автоблокировки (с синхронным кэшом для мгновенного решения на возврате из фона) |
| `pin_lock.dart` | Код-пароль: SHA-256 + соль, антиперебор с паузой |
| `net_diagnostics.dart` | Пошаговые сетевые проверки и вердикт |
| `lan_scanner.dart` | Сканер подсети (TCP + mDNS/Bonjour через `nsd`), семафор на сокеты |
| `analytics.dart` | Firebase Analytics/Crashlytics (лениво — не роняет первый кадр) |
| `quick_actions_service.dart` | Ярлыки long-press иконки (Кабинет/Пополнить/Поддержка/Настройки) |
| `update_check.dart` | Проверка новой версии по `pubspec` в main-ветке GitHub (пока не в сторах) |
| `app_info.dart` | Версия приложения из нативного пакета — **единый источник** (pubspec `version:`) |

### Виджеты (`lib/widgets/`)

`privacy_shield` (заслонка в переключателе задач), `cabinet_skeleton` (скелетон
загрузки с шиммером), `pin_pad` (цифровая клавиатура), `ui_kit` (общие карточка,
заголовок раздела, значок).

## Нативные части

### `ios/`
- **Runner** (приложение): `AppDelegate.swift` (мост к часам, фоновое обновление
  `BGTaskScheduler`, регистрация фраз Siri), `BalanceIntents.swift` (интент Siri
  «баланс»), `WatchSync.swift` (обмен с часами), `SceneDelegate.swift`.
- **Shared/** `BalanceCore.swift` — нативный запрос баланса (для виджета/Siri/часов
  без запуска Dart).
- **BalanceWidget/** `BalanceWidget.swift` — виджеты домашнего/блокировочного экрана
  и кнопка в Пункте управления (WidgetKit + App Intents).
- **InterraWatch/** `InterraWatchApp.swift` — приложение Apple Watch (баланс на
  запястье, данные через WatchConnectivity).

### `android/`
- `MainActivity.kt` — хост Flutter.
- `BalanceWidgetProvider.kt` — виджет баланса на домашнем экране (`res/xml/balance_widget_info.xml`).
- `BalanceTileService.kt` — плитка «Быстрых настроек» с балансом.
- Ярлыки long-press задаются **динамически** через `quick_actions` (статического
  `shortcuts.xml` нет — чтобы совпадать с iOS).

### Общие данные нативных частей
Приложение пишет баланс, номер счёта и токен в общий контейнер (iOS — App Group
`group.ru.interra.lkInterra`, Android — `HomeWidgetPreferences`), откуда их читают
виджеты, плитка, Siri и часы.

## Разработка

```bash
flutter pub get
flutter run                         # на подключённом устройстве/эмуляторе
flutter analyze lib test            # статический анализ (должно быть 0 замечаний)
flutter test                        # юнит-тесты (36 шт.)
```

Тесты (`test/`): `phone_test` (нормализация номера), `balance_store_test`
(парс/формат баланса), `billing_api_test` (разбор ответов `bbb`),
`biometric_lock_test` (льготный период), `update_check_test` (сравнение версий).

## Сборка релиза

```bash
flutter build apk     --release     # Android → build/app/outputs/flutter-apk/app-release.apk
flutter build appbundle --release   # Android App Bundle (для Google Play)
flutter build ios     --release     # iOS (нужен macOS + Xcode; подпись — команда в проекте)
```

- **Android-подпись**: с `android/key.properties` (+ `upload-keystore.p12`) —
  релизным ключом; без него (CI, чужая машина) — debug-ключом, чтобы сборка не
  падала. Оба файла в git не попадают.
- **iOS-подпись**: автоматическая, команда разработчика задана в Xcode-проекте.
  Push включается отдельно на платном аккаунте — см. [`../docs/IOS_PUSH_SETUP.md`](../docs/IOS_PUSH_SETUP.md).
- Иконки и заставка генерируются из `assets/` (`flutter_launcher_icons`,
  `flutter_native_splash`).

Готовность к сторам — [`../docs/APP_STORE_READINESS.md`](../docs/APP_STORE_READINESS.md),
[`../docs/STORE_CHECKLIST.md`](../docs/STORE_CHECKLIST.md).
